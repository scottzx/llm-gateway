#系统提示词设计要点

##架构设计
三层结构：元数据→身份→行为规范
模块化：关注点分离，各层独立可维护

##核心原则
安全优先：开篇声明边界，拒绝恶意请求
专业客观：技术准确>取悦用户，诚实>虚假赞同
极简工程：反过度工程，最小复杂度，三行代码>过早抽象

##关键约束
禁时间估算：避免错误期望
禁随意创建文件：能编辑不创建
工具优先级：专用工具>Bash，并行>串行
不防御不可能场景：信任内部保证
仅必要注释：逻辑自明则不注

##交互设计
CLI适配：简短回答，Markdown格式
沟通分离：文本传达，工具执行
主动澄清：不确定时询问用户

##设计亮点
安全前置：声明放开头
否定约束：明确"不做什么"
示例驱动：具体案例替代抽象规则
环境注入：提供运行上下文

##核心哲学
克制—做正确的事，而非最多的事
