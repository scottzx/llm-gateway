# 日志迁移脚本

将现有的 JSON Lines 格式日志文件迁移到 SQLite 数据库。

## 功能特性

- ✅ 自动扫描 `logs/` 目录下的所有 `.jsonl` 文件
- ✅ 批量导入，性能优化（默认每批 1000 条）
- ✅ 自动去重（根据时间戳、路径、耗时判断）
- ✅ 智能 Token 提取（支持 OpenAI 格式和 SSE 流式格式）
- ✅ 详细的进度显示和错误处理
- ✅ 迁移完成后显示统计信息

## 使用方法

### 标准迁移

```bash
npm run migrate
```

### 自定义批量大小

```bash
# 每批处理 500 条记录
npm run migrate:batch

# 或者自定义
BATCH_SIZE=2000 node scripts/migrate-logs.js
```

### 配置环境变量

在 `.env` 文件中配置：

```env
# 日志文件目录（默认: logs）
CHAT_LOG_DIR=logs

# 数据库文件路径（默认: ./data/chat-logs.db）
DB_LOG_PATH=./data/chat-logs.db

# 批量导入大小（默认: 1000）
BATCH_SIZE=1000
```

## 迁移流程

1. **初始化数据库** - 创建表结构和索引
2. **扫描日志文件** - 读取 `logs/` 目录下所有 `chat-*.jsonl` 文件
3. **解析和去重** - 解析 JSON Lines，检查是否已存在
4. **批量导入** - 以批量方式插入数据库，提升性能
5. **统计报告** - 显示迁移结果和数据库状态

## 注意事项

### 去重逻辑

迁移脚本会根据以下条件判断是否为重复记录：
- 相同的时间戳（精确到秒）
- 相同的请求路径
- 相似的请求耗时（误差 < 10ms）

如果记录已存在，将自动跳过，不会重复导入。

### Token 提取

脚本会智能提取 Token 使用情况：

1. **OpenAI 标准格式**：从 `responseBody.usage` 中提取
2. **SSE 流式格式**：解析 `event: message_delta` 事件
3. **估算方式**：如果没有精确数据，将记录为 0

### 错误处理

- 跳过无法解析的 JSON 行
- 记录导入失败的条目数量
- 不会因为单条记录失败而中断整个迁移

## 重新迁移

如果需要重新导入所有数据（包括已存在的记录）：

1. 删除数据库文件：
   ```bash
   rm data/chat-logs.db
   ```

2. 重新运行迁移：
   ```bash
   npm run migrate
   ```

## 性能优化建议

- **小内存环境**：设置 `BATCH_SIZE=500` 或更小
- **高性能服务器**：设置 `BATCH_SIZE=5000` 或更大
- **SSD 存储**：可以适当增大批量大小

## 故障排查

### 数据库锁定错误

如果遇到 "database is locked" 错误：
1. 确保没有其他进程正在使用数据库
2. 检查是否有 WAL 文件残留
3. 删除 `-wal` 和 `-shm` 文件后重试

### 内存不足

如果遇到内存不足：
1. 减小 `BATCH_SIZE` 值
2. 分批处理日志文件（移动部分文件到临时目录）

### 编码问题

如果遇到中文乱码：
- 确保日志文件是 UTF-8 编码
- 使用 `file` 命令检查文件编码
